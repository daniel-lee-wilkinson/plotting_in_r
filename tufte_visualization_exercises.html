Tufte Visualization Principles: Practice Exercises

Author

Daniel Wilkinson

Introduction

This document provides practice exercises in creating data visualizations that adhere to Edward Tufte’s principles of effective graphical design. Tufte’s principles emphasize:

  1. Data-ink ratio: Maximize the proportion of ink devoted to displaying actual data
  2. Minimize chartjunk: Remove unnecessary decorative elements
  3. Show data variation, not design variation: Let the data speak for itself
  4. Use small multiples: Facilitate comparison through repetition
  5. Integrate text and graphics: Labels should be close to the data they describe
  6. Avoid misleading representations: Use appropriate scales and proportions

Dataset Description

I’ll be working with a sample sales dataset (sample_data.csv) containing:

  • year: Year of observation (2020-2023)
  • region: Geographic region (North, South, East, West)
  • product: Product category (Product_A, Product_B, Product_C)
  • revenue: Sales revenue in dollars
  • units_sold: Number of units sold
  • customer_satisfaction: Customer satisfaction score (1-5 scale)
  • market_share: Market share percentage

Exercise 1: High Data-Ink Ratio Line Chart

Task: Create a line chart showing revenue trends over time for each product, applying Tufte’s principle of maximizing the data-ink ratio.

Requirements: - Remove unnecessary grid lines - Minimize or remove axis lines where appropriate - Use direct labels instead of a legend - Remove background shading - Keep only essential elements

# YOUR CODE HERE
data <- data |>
   mutate(product = gsub("_", " ", product)) 
data |>
  mutate(region = factor(region, levels = c("North", "East", "South", "West"))) |>
  ggplot(aes(x = year, y = revenue, color = product)) +
  geom_line() +
  facet_wrap(~ region) +
  geom_text(
    data = data |> filter(year == max(year)),
    aes(label = product),
    hjust = -0.1, size = 3
  ) +
      scale_colour_manual(values = c("Product A" = "grey20",
                                 "Product B" = "grey60",
                                 "Product C" = "#4a90a4")) +
  coord_cartesian(clip = "off") +
  theme(
    panel.grid       = element_blank(),
    panel.border     = element_blank(),
    axis.line        = element_line(colour = "black", linewidth = 0.3),
    panel.background = element_blank(),
    strip.background = element_blank(),
    legend.position  = "none",
    plot.margin      = margin(r = 60)
  ) +
  labs(x = "Year", y = "Revenue (USD)", 
       title = "Product C leads revenue in every region")

Expected Output: A clean line chart with minimal non-data ink, where product lines are clearly labeled and trends are easy to identify.

Exercise 2: Minimal Bar Chart

Task: Create a bar chart comparing total revenue by region for 2023, following minimalist design principles.

Requirements: - Remove grid lines - Remove chart borders - Use subtle colors or grayscale - Label bars directly (no legend needed) - Remove unnecessary axis elements

Exercise 4: Dot Plot Instead of Bar Chart

Task: Replace a traditional bar chart with a dot plot (Cleveland dot plot) to show market share by product in 2023. Dot plots often have better data-ink ratios than bar charts.

Requirements: - Use points instead of bars - Include reference lines only where they add value - Sort products by market share for easier reading - Remove unnecessary elements

# YOUR CODE HERE
data |>
  filter(year == 2023) |>
  arrange(region, market_share) |>
  mutate(product = factor(product, levels = unique(product))) |>
  ggplot(aes(x = market_share, y = product)) +
  geom_segment(aes(x = min(market_share) -1 , xend = market_share, yend = product),
               colour = "grey80", linewidth = 0.3) +
  geom_point(size = 3, colour = "#4a90a4") +
geom_text(aes(label = paste0(round(market_share, 1), " %")),
          hjust = -0.5, size = 3) +
coord_cartesian(clip = "off") +
theme(plot.margin = margin(r = 20)) +
  facet_wrap(~ region, labeller = labeller(region = toupper)) +
  theme(
    panel.grid       = element_blank(),
    panel.border     = element_blank(),
    panel.background = element_blank(),
    strip.background = element_blank(),
    axis.line        = element_blank(),
    axis.ticks       = element_blank(),
    axis.title.x     = element_blank(),
    axis.text.x      = element_blank()) + 
theme(panel.spacing.y = unit(0.5, "cm")) +
  labs(y = NULL, title = "Product C leads market share in every region, 2023")

Expected Output: A dot plot that efficiently displays market share with minimal ink.

Exercise 8: Range Frame Instead of Full Box

Task: Create a scatter plot using Tufte’s “range frame” concept, where axis lines only extend to the data range rather than creating a full box.

Requirements: - Use theme_tufte() from ggthemes or create custom theme - Show only the data range on axes - Remove unnecessary grid lines - Focus attention on the data points

ggplot(data, aes(x = units_sold, y = revenue)) +
  geom_point(colour = "grey40", size = 2, alpha = 0.7) +
  geom_rangeframe() +
  scale_y_continuous(
    breaks = c(min(data$revenue), max(data$revenue)),
    labels = scales::dollar
  ) +
  scale_x_continuous(
    breaks = c(min(data$units_sold), max(data$units_sold)),
    labels = scales::comma
  ) +
    geom_smooth(method = "lm", se = FALSE, 
            linewidth = 0.3, colour = "grey60") +
  theme_tufte() +
  labs(x = "Units sold", y = "",
       title = "Revenue scales (USD) with units sold")

Expected Output: A plot with minimal framing that draws attention to the data.

Reflection Questions

  1. How does reducing non-data ink change the effectiveness of your visualizations?
    • reduced visual distractions
    • quicker to determine the key message
  2. In what situations might you deviate from strict Tufte principles, and why?
    • when memorability or engagement (through e.g. colour) are priorities
    • removing gridlines, for example, can stop you from seeing exact values
    • to add contextual elements like trend lines
  3. How do small multiples facilitate comparison better than overlaid elements?
    • reduce clutter and overlapping
    • it is easier to isolate patterns in the data
    • enforces consistent scales
    • highlight outliers and trends
  4. What challenges did you encounter when trying to maximize the data-ink ratio?
    • it requires many manual code adjustments, especially to the theme. This means that plotting according to tufte principles cannot be easily automated because of the difference in meaning/intention even between two plots of the same type and data types.
